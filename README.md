(1)程式的功能與技術原理：
這個程式擁有兩個功能，第一個功能是反應式字典。你可以選擇一個純物質，在選擇之後，這個程式會跑出各種包含這個物質的反應式，並且每個反應式有其專屬的編號，當你選擇
編號之後，即可看到這一個反應的詳細資訊。另外，第二個功能叫做"from A to B"，利用功能一的結果，選取一個起點物質，再選取一個終點物質，能夠跑出起點元素加什麼物質
，經過什麼反應(可能有數個反應)，能得到終點物質。而這一個功能會優先評估反應次數最短的一條路徑，盡可能地減少反應的次數。

(2)使用方式：
利用輸入1、2、3選擇模式一、模式二、或是離開程式
功能一：
1.輸入一個物質(程式跑出各反應)
2.選擇編號
3.獲得反應詳細資訊
功能一：
1.輸入一個起點物質
2.輸入一個終點物質
3.獲得最短反應路徑及其相關反應詳細資訊

(3)程式的架構：
Database：
儲存所有化學反應資訊，包括反應物、生成物、反應性質、反應條件等。使用字典，每個化學物質對應多個反應。每個反應有其屬性欄位，包含化學方程式、反應類型（沉澱、酸鹼
、氧化還原等、反應條件（溫度、光照、催化劑等，可選）、可逆性
主程式：
function1：
1.使用 input() 取得使用者要查詢的純物質名稱
2.接著檢查使用者輸入的物質是否存在於資料庫
3.並且取得該物質的所有反應
4.使用迴圈列出每個反應編號與化學方程式
5.使用者輸入要查看的反應編號
6.做 try/except 防止非整數輸入
7.根據使用者選擇，抓取對應的反應資料
8.列出：化學方程式、反應性質、反應條件（若有）、是否可逆
function2：
1.將化學方程式文字拆解成反應物或生成物
2.將化學反應資料庫轉換成圖結構(每個化學物質是節點，每個反應是邊，邊上包含反應式、性質、條件、可逆性)
3.使用 BFS（廣度優先搜尋）找到從起始物質到目標物質的反應路徑
4.建立反應圖，並且呼叫路徑搜尋函數，將找到的路徑用易讀方式列出，包括每一步的反應式、性質、條件、可逆性
main：
1.用 while/True建立一個持續運行的迴圈，直到使用者選擇離開
2.顯示三個選項：
查詢化合物反應（對應 function1）
從 A 到 B 的轉換路徑搜尋（對應 function2）
離開程式
3.根據輸入的數字對應函數：
"1" → function1()
"2" → 先取得起點物質 A 和終點物質 B，再呼叫 function2(start, target, chem_reactions)
"3" → 結束迴圈，結束程式
若輸入無效，給予提示。

(4)開發過程：
利用LLM寫程式，製作過程自己來
1.製作輸入化合物的程式
2.製作data base
3.寫輸入化合物、編號可知化學反應相關性質的程式
4.再一次擴展data base，加入property
5.寫function2
6.擴展data base，加入conditions、reversible
7.debug
8.測試並修改直至成功

(5)開發想法：
國中時我總想說能不能拿一些有害物質來發電(雖然好像現在學到不太可能)，那時的我發現，這個世界上有太多太多的化學反應。如果有一個字典，讓我輸入特定的物質，就能跑出跟這個物質有關的反應，這樣我就輕鬆的把A變成B(例如A當CO2，B當H2O，把CO2變成H2O就能利用水蒸氣來發電)，因此我說利用LLM的力量和程式語言，完成這一個程式。

(6)製作時的挫折：
在製作function2時，我發現程式的結果很常失敗，在探究之後，我發現我的database資料太少導致無法執行成功。而在尋找database的時候，我又遇到了手動查詢耗時過久的問題。一開始，我無從尋找大量的database，並且在每一個化學反應當中，我都要找到他的一些相關的性質，若是用手工查的就會耗費過多時間。隨後，我改變了方法，先利用LLM生成反應式及資料，再去證實資料的可信度，及手動修改AI所產生的錯誤。如此方法使我能夠快速產生巨量的database，function2也才能成功連結。
